# Product Plan

## Vision

Build a neutral public accountability platform where people can report local problems, verify evidence, track solutions, and compare public promises with actual progress.

## Core rule

The platform records civic evidence. It does not campaign for or against any party, leader, candidate, or ideology.

## MVP sections

### 1. My Area

Users select state, district, city, village, ward, or block. The feed shows public problems from that area.

Each problem should have:

- Title
- Category
- Location
- Evidence
- Status
- Timeline
- Reporter trust score
- Duplicate report check

### 2. National

Large issues can be promoted to national visibility after verification.

Examples:

- Water crisis
- Pollution
- Unemployment
- Public health shortage
- Infrastructure failure

### 3. Solved

Solved records should show:

- What the problem was
- When it was reported
- How it was verified
- What changed
- Proof after resolution

Avoid turning solved records into political promotion.

### 4. Leaders

Leader profiles should be data-weighted. Public review should not dominate the score.

Suggested score:

- 40% issue resolution rate
- 20% promise completion
- 15% response time
- 10% transparency
- 10% verified public feedback
- 5% public conduct/accessibility

## Risky feature changes

Avoid names like `Danger List`. Use neutral terms:

- Accountability Watch
- Low Performance Index
- Public Concern List
- Performance Alert

## Future modules

- Election promise tracker
- Public budget tracker
- Civic office response dashboard
- Reporter trust score
- Evidence verification queue
- False report penalties
- Area heat map
- Public solution suggestions
